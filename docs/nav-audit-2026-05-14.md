# Navigation Bar Audit — 2026-05-14

## Scope
- Audited all Next.js page routes under `app/**/page.tsx` (excluding `app/api/**`).
- Requirement checked: every page should render the top navigation bar except kiosk pages (`/kiosk/*`).
- Validation method used:
  1. Static scan for direct `<TopNav />` usage.
  2. Static scan of ancestor `layout.tsx` files for `<TopNav />`.
  3. Manual review of special pages (redirect-only, print-only, TV/driver flows).

## High-level findings
- Total pages scanned: **104**.
- Kiosk pages excluded from requirement: **1** (`app/kiosk/[branch]/page.tsx`).
- Pages with nav confirmed either directly or via layout: majority of app pages.
- Pages intentionally outside normal app shell (currently **no TopNav**):
  - Login / auth redirect pages.
  - TV board pages.
  - Driver mobile pages.
  - Dispatch POD capture and printable run sheet pages.
  - Lightweight redirect-only pages.

## Pages that violate the strict requirement as written
If we apply the rule literally (only kiosk exempt), these routes are non-compliant because they do not render `<TopNav />`.

### Authentication
- `/login` → `app/login/page.tsx`
- `/ops-login` (redirects to `/login`) → `app/ops-login/page.tsx`

### TV / Driver / specialized operational flows
- `/tv/[branch]` → `app/tv/[branch]/page.tsx`
- `/driver` → `app/driver/page.tsx`
- `/driver/route/[id]` → `app/driver/route/[id]/page.tsx`
- `/dispatch/pod/[so]` → `app/dispatch/pod/[so]/page.tsx`
- `/dispatch/run-sheet/[routeId]` → `app/dispatch/run-sheet/[routeId]/page.tsx`

### Redirect-only pages (no UI shell)
- `/sales/deliveries` redirects to `/sales/tracker` → `app/sales/deliveries/page.tsx`
- `/sales/history` redirects to `/sales/transactions?tab=history` → `app/sales/history/page.tsx`

## Notes on likely intentional exceptions
These pages appear intentionally designed without the standard nav:
- `/login` and `/ops-login`: authentication flow.
- `/tv/[branch]`: wallboard-style display.
- `/driver*` and `/dispatch/pod/*`: focused mobile/operator views.
- `/dispatch/run-sheet/*`: print-friendly document view.
- Redirect-only pages have no rendered page chrome by design.

## Refactor opportunities
1. **Create explicit route-group shell contracts**
   - Add route groups like `(app-shell)` and `(no-shell)` with dedicated layouts.
   - Move pages that should always show nav into `(app-shell)` where `TopNav` is guaranteed once.
   - Keep kiosk, TV, login, print, and driver pages in `(no-shell)` for explicit exception handling.

2. **Stop duplicating `TopNav` in individual pages/clients**
   - Many pages mount `TopNav` directly in page/client components.
   - Consolidate into sectional layouts (e.g. warehouse, dispatch, sales, services) to remove repeated boilerplate and avoid drift.

3. **Single source of truth for menu rendering**
   - `src/lib/menu-config.ts` exists but `TopNav` still contains hardcoded menu rendering.
   - Wire `TopNav` to `visibleMenu()` so menu permissions and links are centrally managed.

4. **Add automated guardrail**
   - Add a CI script that scans `app/**/page.tsx` and fails when a non-exempt route lacks nav via direct render or parent layout.
   - Keep an explicit allowlist for exempt route patterns (`/kiosk/*`, `/login`, `/tv/*`, etc.) to avoid accidental regressions.

## Proposed implementation plan
- Phase 1: Agree on final exception list (strictly kiosk-only vs operational exceptions).
- Phase 2: Introduce `(app-shell)` layout with `<TopNav />` and migrate standard app routes.
- Phase 3: Move specialized pages into `(no-shell)` and codify this in docs + CI nav-lint script.
- Phase 4: Refactor `TopNav` to consume `src/lib/menu-config.ts` for consistent RBAC-driven menus.
